# Hirely

An AI-powered resume screening platform that intelligently ranks candidates using transformer-based semantic similarity, NLP skill extraction, and explainable AI insights.

---

## Features

- **Semantic ranking** вЂ” resumes are scored against a job description using sentence-transformer embeddings (`all-MiniLM-L6-v2`), so meaning is understood rather than just keyword overlap
- **Weighted Compatibility Score** вЂ” a 100-point score combining semantic similarity (50 pts), skill match (20 pts), experience (15 pts), education (10 pts), and certifications (5 pts)
- **Skill gap analysis** вЂ” matched and missing skills extracted with spaCy NLP
- **AI insights** вЂ” candidate strengths and tailored interview questions generated by Google Gemini, with an automatic rule-based fallback if no API key is configured
- **Parallel processing** вЂ” multiple resumes are screened concurrently using asyncio worker threads
- **JWT authentication** вЂ” user accounts with bcrypt password hashing and token-based sessions
- **Resume file serving** вЂ” authenticated PDF/DOCX retrieval with path-traversal protection
- **Persistent storage** вЂ” jobs and screening results stored in SQLite via SQLAlchemy

## Getting Started

### Prerequisites

- Python 3.13+
- Node.js 18+

### Backend

```bash
cd backend

# Create and activate a virtual environment
python -m venv venv
venv\Scripts\activate          # Windows
# source venv/bin/activate     # macOS/Linux

# Install dependencies
pip install -r requirements.txt

# Download the spaCy model (if not already installed)
python -m spacy download en_core_web_sm

# Start the API server
uvicorn app.main:app --reload
```

The API runs at `http://127.0.0.1:8000`. The SQLite database and upload directory are created automatically on first startup.

### Frontend

```bash
cd frontend
npm install
npm run dev
```

The app runs at `http://localhost:5173`.

### Optional: Gemini AI Insights

Create `backend/gemini_api.env` with your Google Gemini API key:

```env
GEMINI_API_KEY=your_key_here
```

Without this file, the platform still works fully вЂ” strengths and interview questions fall back to the built-in rule-based generator.

## API Reference

| Method | Endpoint | Auth | Description |
|---|---|---|---|
| POST | `/auth/register` | No | Create a new account |
| POST | `/auth/login` | No | Log in and receive a JWT |
| GET | `/health` | No | Health check |
| POST | `/screening/analyze` | Yes | Upload resumes + JD, get ranked results |
| GET | `/screening/jobs` | Yes | List past screening jobs |
| GET | `/screening/results/{job_id}` | Yes | Retrieve results for a job |
| GET | `/screening/resume/{candidate_id}` | Yes | Download/preview a candidate's resume |

Protected routes require `Authorization: Bearer <token>` header.

## Scoring Model

Each resume receives a **Candidate Compatibility Score** out of 100:

| Component | Max Points | Method |
|---|---|---|
| Semantic similarity | 50 | Cosine similarity of sentence-transformer embeddings |
| Skill match | 20 | Fraction of JD skills found in resume (spaCy extraction) |
| Experience | 15 | Regex year-extraction vs. required years in JD |
| Education | 10 | Keyword detection (degree, university, etc.) |
| Certifications | 5 | Keyword detection (certified, projects, etc.) |

**Recommendation tiers:**

| Score | Tier |
|---|---|
| в‰Ґ 58 | Strong Match |
| в‰Ґ 45 | Good Match |
| в‰Ґ 38 | Average Match |
| < 38 | Weak Match |
